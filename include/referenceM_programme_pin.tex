\paragraph{The \texttt{referenceM}$=4$ programme pin.}
The lock-in shell index $\mathrm{referenceM}=4$ is \emph{not} a per-channel fit knob.
Lean defines it as $\mathrm{referenceM}=\texttt{qcdShell}+\texttt{stepsFromQCDToLockin}$
with the current substrate pins $\texttt{qcdShell}=1$ and $\texttt{latticeStepCount}=3$
(\leanfile{Hqiv/Geometry/OctonionicLightCone.lean};
theorem \leanid{referenceM_eq_four}).
It is the \textbf{first horizon row with enough octonion-mode degrees of freedom}
to host the full sector spine: at $m=4$ the library proves
$N_{\mathrm{new}}(4)=40$ (\leanid{new_modes_referenceM_numeric}), and the
machine-checked closure $G_2\cup\{\Delta\}\Rightarrow\mathfrak{so}(8)$
(\leanfile{Hqiv/Algebra/SO8ClosureAbstract.lean},
\leanid{G2_plus_Delta_closes_to_so8}) supplies the fourteen $G_2$ derivation
generators plus harmonic $\Delta$ on the eight-dimensional carrier.
Three generations remain a Spin(8)/triality readout (\leanfile{Hqiv/Algebra/Triality.lean}),
not a consequence of the numeral~$4$ itself.

Changing \texttt{qcdShell} or \texttt{latticeStepCount} moves which grid row is
named ``reference''; it does not alter the algebraic fact that generation space
is threefold.
The numeral~$4$ also coincides with $\texttt{tuftHeavyChartShell}=n+1$ for heavy
Hopf winding $n=3$, but Lean records that equality only as a numeric coincidence
under the current pins, not as chart identification
(\leanfile{Hqiv/Physics/HopfShellBeltramiMassBridge.lean}).
