\section*{Current Lean build split}
\begin{sloppypar}
The repository exposes the following \emph{named} Lake targets (see \path{lakefile.toml} for the authoritative list):
\begin{itemize}
\item \texttt{lake build HQIVPaperClaims}: lightweight target for \path{papers/closure.tex} using \path{Hqiv/Geometry/OctonionicLightCone.lean} and \path{Hqiv/SO8ClosureSymbolic.lean} only (no satisfiability formalism in this mirror; no heavy matrix-certificate elaboration).
\item \texttt{lake build HQIVSO8Closure}: heavyweight matrix-closure target (\path{Hqiv/GeneratorsLieClosureData*.lean}, \path{Hqiv/GeneratorsLieClosure.lean}, \path{Hqiv/LieBracketCell.*}, \path{Hqiv/Algebra/SO8ClosureAbstract.lean}). Lie-bracket coefficients are elaborated module-by-module; the linear-independence step in \path{Hqiv/GeneratorsLieClosure.lean} routes through the single Gram-matrix axiom \texttt{so8CoordMatrix\_transpose\_mul\_self} in \path{Hqiv/So8CoordMatrix.lean} (documented bridge to the numeric orthonormality script). Expect a long cold-cache build. \emph{RAM:} default parallel Lake builds of the 784 \path{LieBracketCell} modules can exceed \(\sim\)100GB; use a full repository checkout and \texttt{scripts/build\_hqiv\_so8\_closure\_lowmem.sh} (\texttt{LEAN\_NUM\_THREADS=1}, \texttt{lake build \ldots\ -j 1}). The frozen \path{paper\_refs\_bundle\_2026-05-06/} mirror omits those \texttt{.lean} sources and supports \texttt{lake build HQIVPaperClaims} only; see \path{Hqiv/LieBracketCell/README.md} inside the bundle.
\item \texttt{lake build HQIVStrongColorSu3Certificate}: optional \(\mathfrak{su}(3)\) color chart certificate (auto-generated \texttt{@[simp]} atoms for nonzero \(f^{abc}\) in \path{Hqiv/Physics/StrongColorSu3fStructureSimp.lean}; not part of the default \texttt{HQIVLEAN} glob).
\item \path{Hqiv/Algebra/WeakFromLeftMulOctonion.lean}: electroweak-style \emph{definitions} from $L(e_i)$ (complex-structure $J$, candidate $W^\pm$, $Z$) with proved skewness only; \(\mathfrak{su}(2)\) closure for weak isospin remains in \path{Hqiv/Algebra/SMEmbedding.lean}.
\end{itemize}
The SO(8) paper targets and their referenced files are included in \PaperReproBundleDir/; the SU(3) certificate target is built in-repo on demand (and in CI) but is not duplicated in this frozen bundle.
